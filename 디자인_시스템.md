## 📐 레이아웃 규칙 (윈도우 데스크톱 - 코디네이터용)

### 1. Split View 구조
- **좌측 (70%):** 지도 영역 (반응형 높이)
- **우측 (30%):** 매칭 패널 (스크롤 가능)
- **상단 (전체 너비):** AppBar (상태 정보 표시)

### 2. 상단 AppBar (높은 가시성)
- **좌측 섹션:**
  - 큰 로고: "LMP-Link"
  - 작은 글씨 부제목: "장애인 활동지원 매칭 콘솔"
  
- **중앙 섹션:**
  - 현재 날짜/시간
  - 빠른 통계: "오늘: 12건 처리 / AI 대기 2건"
  
- **우측 섹션:**
  - 로그인한 코디네이터 이름
  - Dev/Prod 환경 토글 (명확하게 표시)
  - 로그아웃 버튼

### 3. 지도 영역 (좌측 70%)
- **네이버 지도 전체 뷰**

- **우상단 플로팅 컨트롤 박스:**
  - 반경 프리셋 버튼: `[1km] [3km (기본 강조)] [5km]`
  - 레이어 토글: `[이용자 보기 ON/OFF] [지원사 보기 ON/OFF]`

- **좌상단 검색 박스:**
  - 입력: "이용자 이름/ID 검색"
  - 버튼: "검색" (청록색)

- **마커 스타일:**
  - **이용자:** 굵은 테두리의 파란 원, 약간 큼 (30x30px)
  - **지원사:** 주황 사각형, 동일 크기 (24x24px)
  - **클러스터링:** 줌 아웃 시 흰색 숫자가 있는 반투명 버블

- **원클릭 반경 기능 (핵심 상호작용):**
  - 이용자 마커 클릭 → 즉시 3km 원 그려짐
  - 지도 하단에 표시: "반경: 3km / 지원사 7명 발견"
  - 우측 패널 자동 필터링 및 스크롤 이동

### 4. 우측 HUD 패널 (30%)
- **탭 네비게이션 (상단):**
  - `[이용자 목록] [매칭 후보 / AI 추천]`

#### 탭 1: 이용자 목록
- **각 항목 (큼, 스크롤 가능 리스트):**
  - 이용자 이름 (굵게, 16pt+)
  - 주소 요약 (시/구만)
  - 주요 요구사항 (1줄)
  - 현재 상태 배지 (미매칭 / 진행중 / 완료)
  - 클릭 시: 지도에 강조 + 자동 3km 반경 + 탭 2로 이동

#### 탭 2: 매칭 후보 / AI 추천
- **상단 필터 바 (크고 명확한 컨트롤):**
  - 시간대 드롭다운 (평일 오전 / 오후 / 주말)
  - 성별 토글 (전체/남/여)
  - 자차 보유 토글 (포함/제외)
  - 기피 대상 제외 체크박스
  - "필터 초기화" 링크 (우측)

- **후보 리스트 (필터 아래):**
  - 각 지원사 카드 형태:
    - 이름, 거리 (예: "2.1km")
    - 경력 요약, 시간대 호환성 아이콘 (초록 체크 / 노랑 느낌표)
  - 우측에 "가매칭" 버튼 (큰 pill 형태)

- **패널 상단/중앙 굵은 AI 추천 버튼:**
  - **비활성 상태 (선택 전):** 회색, "먼저 후보를 선택해 주세요"
  - **활성 상태 (1명 이상 선택):** 강조 색(청록), "AI 추천 분석 실행"
  - 클릭 시:
    - 얇은 ProgressBar (상단에 표시)
    - AI 분석 완료 시, 상단에 "AI 추천 TOP 3" 섹션 생성
    - 각 카드에 추천 점수 배지 표시 (예: "92점")

## 📱 모바일 규칙 (Android - 활동지원사용)

### 1. 네비게이션 구조: 하단 3탭
- `[오늘 일정] [알림] [내 정보]`

### 2. 탭: 오늘 일정
- **상단 섹션:**
  - 큰 날짜 표시: "오늘, 화요일 12월 9일"
  - "오늘 할 일 2건"

- **일정 카드 (스크롤 가능):**
  - 시간 (예: "10:00–12:00")
  - 이용자 이름
  - 주소 (동/구까지)
  - 간단한 요구사항 (1줄)
  - 카드 색상: 오늘=밝음 / 미래=회색 / 완료=흐림

- **탭 시:**
  - 상세 페이지:
    - 전체 주소 및 "지도에서 보기" 버튼 (네이버 맵 앱 실행)
    - 담당 코디네이터 연락처 (크고 명확한 전화 버튼)
    - 특이사항 (텍스트)

### 3. 탭: 알림
- **상단 배지:** "새 가매칭 제안 있음"
- **항목 형식:**
  - "[새 가매칭] 홍길동 님 – 주 3회 오후 / 강남구"
  - 버튼: [검토] [거절]
  - "매칭 확정 안내" 등 상태 변경 알림

- **[검토] 클릭:**
  - 매칭 상세 페이지로 이동:
    - 일정, 위치, 요구사항 간단 요약
    - "수락"과 "거절" 버튼만 배치 (크게)

### 4. 탭: 내 정보
- **프로필 섹션:**
  - 프로필 사진 (선택)
  - 이름
  - 자격/경력 태그

- **설정:**
  - 희망 근무 시간대 (토글)
  - 선호 지역
  - 기피 대상 체크 설정
  - 저장 버튼 (하단 고정, 크게)

### 5. 모바일 접근성 규칙
- **폰트:** 굵고 선명한 한글 폰트 (Apple SD Gothic, Roboto)
- **본문 크기:** 16~18pt
- **버튼 텍스트:** 최소 18~20pt
- **터치 타깃:** 최소 48x48px
- **여백:** 버튼 주변 충분히 확보

## 🧩 핵심 컴포넌트 상세 사양

### 컴포넌트 1: HUD 패널 (우측)
- **목적:** 실시간 매칭 운영 인터페이스
- **구조:**
  - 상단: 탭 + 필터 컨트롤
  - 중간: 스크롤 가능 카드 리스트
  - 상단 고정: AI 추천 버튼 & 결과
- **상호작용:**
  - 필터 적용 → 리스트 100ms 내 업데이트
  - 카드 클릭 → 지도 강조
  - AI 버튼 → 로딩 상태 → 결과 표시

### 컴포넌트 2: 비교 모달 (상세보기)
- **트리거:** "가매칭" 버튼 또는 리스트 더블클릭
- **레이아웃:**
  - 헤더: "매칭 검토" + 닫기 + 저장 버튼
  - 본문:
    - 좌측 컬럼: 이용자 정보 카드
    - 우측 컬럼: 지원사 정보 카드
    - 중앙 상단: 매칭 적합도 바 (0-100, 초록 그라데이션)
    - 중앙: 3개 작은 아이콘 (거리 / 시간대 호환 / 성향)
  - 하단: AI 추천 사유 (3줄 자연스러운 한글)

### 컴포넌트 3: AI 인사이트 박스
- **스타일:** 별/스파클 아이콘이 있는 구분 카드
- **내용:**
  - 제목: "AI 추천 사유"
  - 본문: 3~4줄 짧고 쉬운 한글
  - 액션: "[의견 다시 생성]" 링크

### 컴포넌트 4: 마커 스타일
- **이용자 마커 (파란 원):**
  - 배경: #3B82F6 (파랑)
  - 아이콘: 흰색 사람 실루엣
  - 테두리: 굵은 흰색 (2~3px)
  - 크기: 32x32px

- **지원사 마커 (주황 사각형):**
  - 배경: #F97316 (주황)
  - 아이콘: 흰색 브리프케이스/서류
  - 테두리: 얇은 흰색 (1px)
  - 크기: 24x24px

### 컴포넌트 5: 클러스터 마커
- **스타일:** 흰 배경의 둥근 버블
- **내용:** 큰 흰색 숫자 (18pt+)
- **배경:** 반투명 회색 (#64748B, 불투명도 80%)
- **크기:** 클러스터 크기에 따라 적응

## 🎯 상호작용 흐름

### 흐름 1: "근처 지원사 찾기" (메인 워크플로우)
1. 사용자가 지도의 **파란 원(이용자 마커)** 클릭
2. **3km 원** 즉시 그려짐
3. **우측 패널 필터링:** 매칭 가능한 지원사 카드 표시
4. 지도에서 반경 **바깥쪽 지원사는 어둡게** (불투명도 50%)
5. 코디네이터가 반경 프리셋 버튼으로 범위 변경 가능
6. 카드 훑어보고, 선택적으로 **AI 분석** 트리거

### 흐름 2: "빠른 매칭" (간편 매칭)
1. 코디네이터가 필터링된 리스트에서 1~3명 지원사 선택
2. **AI 분석** 버튼 클릭
3. **ProgressBar** 표시, 분석 실행 (2~10초)
4. **AI TOP 3 섹션** 리스트 상단에 생성
5. 각 카드에 점수 배지 + 짧은 추천 사유
6. 코디네이터가 "가매칭" 버튼 클릭 → 확인 모달 띄움
7. 확인 시 → `matches` 테이블에 저장 (상태: Draft)

### 흐름 3: "매칭 최종 확정" (관리자 확정)
1. 모달 제목이 "매칭 확정"으로 변경
2. 기존 버튼 "가매칭"이 "확정 및 알림"으로 교체
3. 클릭 시 → 상태를 "Confirmed"으로 변경
4. Realtime 이벤트 발생 → 모바일 앱에 알림 표시

## ✅ 접근성 & 사용성 체크리스트

- [ ] **색상 대비:** 모든 텍스트가 WCAG AA 기준 충족 (본문 4.5:1, 큰 텍스트 3:1)

- [ ] **에러 메시지:** 명확하고 실행 가능한 사용자 언어

- [ ] **다크 모드:** 전체 화면에서 높은 대비 유지

## 🎨 6. MAUI XAML 코드 예시

### 6.1 색상 팔레트 (Colors.xaml)

**파일:** `Resources/Styles/Colors.xaml`

```xml
<?xml version="1.0" encoding="utf-8" ?>
<ResourceDictionary 
    xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
    xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml">

    <!-- Background Colors -->
    <Color x:Key="BackgroundDeepNavy">#111827</Color>
    <Color x:Key="SurfaceDarkGray">#1F2937</Color>

    <!-- Text Colors -->
    <Color x:Key="TextPrimary">#F9FAFB</Color>
    <Color x:Key="TextSecondary">#9CA3AF</Color>
    <Color x:Key="TextTertiary">#6B7280</Color>

    <!-- Action Colors -->
    <Color x:Key="PrimaryCTA">#2DD4BF</Color>
    <Color x:Key="UserMarker">#3B82F6</Color>
    <Color x:Key="AssistantMarker">#F97316</Color>

    <!-- Feedback Colors -->
    <Color x:Key="Success">#10B981</Color>
    <Color x:Key="Warning">#FBBF24</Color>
    <Color x:Key="Error">#EF4444</Color>

    <!-- MAUI Template Colors (Mapped) -->
    <Color x:Key="Primary">#0066CC</Color>
    <Color x:Key="Secondary">#FF9500</Color>
    <Color x:Key="Tertiary">#2E7D32</Color>

</ResourceDictionary>
```

### 6.2 타이포그래피 스타일 (Styles.xaml)

**파일:** `Resources/Styles/Styles.xaml`

```xml
<?xml version="1.0" encoding="utf-8" ?>
<ResourceDictionary 
    xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
    xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml">

    <!-- Display: 로고, 모달 타이틀 -->
    <Style x:Key="DisplayText" TargetType="Label">
        <Setter Property="FontSize" Value="32" />
        <Setter Property="FontAttributes" Value="Bold" />
        <Setter Property="TextColor" Value="{StaticResource TextPrimary}" />
    </Style>

    <!-- Heading 1: 페이지 섹션 -->
    <Style x:Key="Heading1" TargetType="Label">
        <Setter Property="FontSize" Value="24" />
        <Setter Property="FontAttributes" Value="Bold" />
        <Setter Property="TextColor" Value="{StaticResource TextPrimary}" />
    </Style>

    <!-- Heading 2: 카드 타이틀 -->
    <Style x:Key="Heading2" TargetType="Label">
        <Setter Property="FontSize" Value="20" />
        <Setter Property="FontAttributes" Value="Bold" />
        <Setter Property="TextColor" Value="{StaticResource TextPrimary}" />
    </Style>

    <!-- Body: 메인 콘텐츠 -->
    <Style x:Key="BodyText" TargetType="Label">
        <Setter Property="FontSize" Value="16" />
        <Setter Property="TextColor" Value="{StaticResource TextPrimary}" />
    </Style>
```
